浏览器原生 WebRTC 1 对 1 视频通话 Demo
====================================

目录结构：
  index.html   - 前端页面（直接用浏览器打开）
  server.js    - Node.js WebSocket 信令服务器

一、准备环境
----------
1. 安装 Node.js（建议 16+，推荐 18+）
2. 在本项目目录（包含 index.html、server.js 的目录）打开终端或 PowerShell

二、安装依赖（仅首次需要）
----------------------
在本目录执行：

    npm init -y
    npm install ws

三、一行命令启动信令服务器
------------------------

    node server.js

看到控制台输出：

    WebRTC signaling server is running on ws://localhost:8080

说明服务器已就绪。

四、启动前端页面（双击即可）
------------------------
⚠️ 重要：为了能正常调用摄像头/麦克风，建议通过 http://localhost 打开页面，
而不是直接双击（file://）打开。多数浏览器会限制 file:// 的摄像头权限。

1. 启动 server.js 后，用浏览器打开：

    http://localhost:8080

2. 建议在「两台不同设备」或「同一设备的两个浏览器窗口」中打开本页面。

五、进行 1 对 1 通话
------------------
1. 在两端页面中：
   - 输入相同的房间号（例如：webrtc-demo-001）
   - 信令服务器地址保持为：ws://127.0.0.1:8080（同机）或 ws://你的局域网IP:8080（跨设备）
   - 点击「连接 & 加入房间」
2. 任一端点击底部控制栏上的摄像头 / 麦克风按钮，即可开始通话：
   - 大屏：远端视频
   - 右下角小窗：本地视频（带镜像）
3. 点击红色挂断按钮可结束通话。

六、常见问题
----------
1）浏览器提示没有权限访问摄像头/麦克风
   - 请点击地址栏右侧的权限图标，允许当前页面使用摄像头和麦克风。

2）两端看不到对方视频
   - 确认两边输入的房间号完全一致（区分大小写）。
   - 确认 server.js 已经在对应机器上运行，并且浏览器可以访问该 IP/端口。
   - 查看浏览器控制台是否有错误信息。

3）希望局域网中其它设备访问
   - 将信令地址从 ws://127.0.0.1:8080 改为 ws://你的局域网 IP:8080
   - 确认防火墙允许 8080 端口访问。
