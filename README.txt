浏览器原生 WebRTC 多人会议（SFU / mediasoup）
=========================================

目录结构：
  index.html     - 前端页面（通过 http/https 打开）
  src/app.js     - 前端逻辑源码（会被打包）
  public/app.js  - 打包产物（npm start 时自动生成）
  server.js      - Node.js mediasoup SFU + WebSocket 信令 + HTTP 静态服务

一、准备环境
----------
1. 安装 Node.js（建议 18+）
2. 在本项目目录（包含 index.html、server.js 的目录）打开终端或 PowerShell

二、安装依赖（仅首次需要）
----------------------
在本目录执行：

    npm install

三、一行命令启动 SFU 服务器
------------------------

    npm start

看到控制台输出（本地模式）：

    HTTP : http://localhost:8080
    WS   : ws://localhost:8080/ws
    [mediasoup] announcedIp=49.233.175.236 rtcPortRange=49160-49200

说明服务器已就绪。

四、启动前端页面（双击即可）
------------------------
⚠️ 重要：为了能正常调用摄像头/麦克风，建议通过 http://localhost 打开页面，
而不是直接双击（file://）打开。多数浏览器会限制 file:// 的摄像头权限。

1. 启动 server.js 后，用浏览器打开：

    http://localhost:8080

2. 建议在「两台不同设备」或「同一设备的两个浏览器窗口」中打开本页面。

五、进行多人会议（最多 16 人）
--------------------------
1. 在两端页面中：
   - 输入相同的房间号（例如：webrtc-demo-001）
   - 信令服务器地址：
       - 本地：ws://127.0.0.1:8080/ws
       - 云端：wss://livebridge.cn/ws
   - 点击「Join Room」
2. 每个加入者会发布自己的音视频，并订阅房间内其他人的音视频（SFU 转发）。
3. 点击红色挂断按钮可离开房间。

六、常见问题
----------
1）浏览器提示没有权限访问摄像头/麦克风
   - 请点击地址栏右侧的权限图标，允许当前页面使用摄像头和麦克风。

2）两端看不到对方视频
   - 确认两边输入的房间号完全一致（区分大小写）。
   - 确认 server.js 已经在对应机器上运行，并且浏览器可以访问该 IP/端口。
   - 查看浏览器控制台是否有错误信息。

3）希望局域网中其它设备访问
   - 将信令地址从 ws://127.0.0.1:8080 改为 ws://你的局域网 IP:8080
   - 确认防火墙允许 8080 端口访问。

4）跨局域网/跨运营商的稳定性
   - 仅 STUN 可能会失败（复杂 NAT/防火墙）。
   - 你已部署 TURN/TURNS：turn.livebridge.cn:3478 / turns:5349
   - 本项目已在前端默认配置了 TURN（账号 webrtc）。
